UniAI CLI 把 Web UI 能做的事几乎全部搬到了命令行上（创建 [issue](/issues)、分配 [智能体](/agents)、启动 [守护进程](/daemon-runtimes) 等等）。这一页把所有顶级命令列出来，每条配一句用途。完整 flag 和示例用 `uniai <命令> --help` 查。

## 认证入口

第一次用 CLI 时先登录，拿一个**个人访问令牌（Personal Access Token，PAT）**：

```bash
uniai login
```

浏览器会自动打开，你在 Web 端同意后，CLI 把 PAT（`mul_` 前缀）保存到 `~/.multica/config.json`。此后所有命令都会自动用这个 PAT 认证。

<Callout type="tip">
CI / 无浏览器环境跳过浏览器流程：先在 Web 端 **Settings → Personal Access Tokens** 创建一个 PAT，然后 `uniai login --token <mul_...>` 直接填入。
</Callout>

Token 类型的详细区分见 [认证与令牌](/auth-tokens)。

## 认证与初始化

| 命令 | 用途 |
|---|---|
| `uniai login` | 登录并保存 PAT |
| `uniai auth status` | 查看当前登录状态、用户、工作区 |
| `uniai auth logout` | 清除本地 PAT |
| `uniai setup cloud` | UniAI Cloud 一键初始化（登录 + 装 daemon） |
| `uniai setup self-host` | 自部署后端的一键初始化 |

## 工作区和成员

| 命令 | 用途 |
|---|---|
| `uniai workspace list` | 列出你有权访问的所有工作区 |
| `uniai workspace get <slug>` | 查看一个工作区的详情 |
| `uniai workspace member list` | 列出当前工作区的成员 |
| `uniai workspace update <id> --name "..." [--description "..."] [--context "..."] [--issue-prefix "..."]` | 修改 workspace 元数据（admin/owner 权限）。长文本可用 `--description-stdin` / `--context-stdin`。 |

## Issue 和 Project

<Callout type="info">
`list` 类命令（`uniai issue list`、`autopilot list`、`project list` 等）表格里默认显示**可直接复制**的短 ID：issue 是 key（如 `UNI-123`），其余资源是 UUID 短前缀。下面表格里的 `<id>` 同时接受短 ID 和完整 UUID，所以典型用法是 `uniai issue list` → 复制 key → `uniai issue get UNI-123`。需要完整 UUID 时给 `list` 加 `--full-id`。
</Callout>

| 命令 | 用途 |
|---|---|
| `uniai issue list` | 列出 issue（默认显示可复制的 issue key） |
| `uniai issue get <id>` | 查看单条 issue（接受 issue key 或 UUID） |
| `uniai issue create --title "..."` | 创建新 issue |
| `uniai issue update <id> ...` | 修改 issue（状态、优先级、分配人等） |
| `uniai issue assign <id> --to <name>` | 分配给成员、智能体或小队（分配给智能体会触发一次运行） |
| `uniai issue status <id> <status>` | 快捷改状态 |
| `uniai issue search <query>` | 关键字搜索 |
| `uniai issue children <id>` | 按 stage 分组查看子 issue |
| `uniai issue pull-requests <id>` | 查看关联 PR 及其状态 |
| `uniai issue runs <id>` | 查看 issue 上智能体跑过的任务 |
| `uniai issue run-messages <task-id>` | 查看某次执行的消息 |
| `uniai issue usage <id>` | 查看单个 issue 聚合 token 用量 |
| `uniai issue rerun <id>` | 给该 issue 当前的智能体分配人重新创建一条任务 |
| `uniai issue cancel-task <task-id>` | 取消排队中或运行中的任务 |
| `uniai issue comment <id> ...` | 嵌套：看 / 发评论 |
| `uniai issue comment resolve/unresolve <comment-id>` | 标记评论线程已解决 / 未解决 |
| `uniai issue subscriber <id> ...` | 嵌套：订阅 / 取消订阅 |
| `uniai issue metadata <id> ...` | 嵌套：读写 issue metadata |
| `uniai issue label <id> ...` | 嵌套：管理 issue 上的 label |
| `uniai project list/get/create/update/delete/status` | Project CRUD |
| `uniai label list/create/update/delete` | Workspace label CRUD |

|---|---|
| `uniai agent list` | 列出工作区的智能体 |
| `uniai agent get <slug>` | 查看智能体配置 |
| `uniai agent create ...` | 创建智能体 |
| `uniai agent update <slug> ...` | 修改智能体 |
| `uniai agent archive <slug>` | 归档 |
| `uniai agent restore <slug>` | 恢复归档的智能体 |
| `uniai agent tasks <slug>` | 查看智能体的任务历史 |

| `uniai skill list/get/create/update/delete` | Skill CRUD |
| `uniai skill import ...` | 从 GitHub / ClawHub / 本机导入 Skill |
| `uniai skill files ...` | 嵌套：管理 Skill 的文件 |

### Skill 导入冲突

`uniai skill import --url <url>` 默认等同于 `--on-conflict fail`。如果工作区里已经有同名 Skill，命令会返回结构化 `conflict` 结果并退出，不会修改工作区。

如果你是已有 Skill 的 creator，并且想用新导入内容覆盖它，同时保留原 Skill 的 ID 和 agent 绑定，用 `--on-conflict overwrite`。如果想保留已有 Skill、另存一份，用 `--on-conflict rename`，系统会自动加 `-2` 这类后缀。如果只是批量导入时遇到同名项就跳过，用 `--on-conflict skip`。

```bash
uniai skill import --url https://skills.sh/acme/repo/review-helper
uniai skill import --url https://skills.sh/acme/repo/review-helper --on-conflict overwrite
uniai skill import --url https://skills.sh/acme/repo/review-helper --on-conflict rename
uniai skill import --url https://skills.sh/acme/repo/review-helper --on-conflict skip
```

## 小队

| 命令 | 用途 |
|---|---|
| `uniai squad list` | 列出工作区里的小队 |
| `uniai squad get <id>` | 查看一个小队 |
| `uniai squad create --name "..." --leader <agent>` | 创建小队（owner / admin）|
| `uniai squad update <id> ...` | 修改名字、描述、instructions、队长、头像 |
| `uniai squad delete <id>` | 归档（软删除）—— 同时把分配给小队的 issue 转给队长 |
| `uniai squad member list/add/remove/set-role <squad-id>` | 管理小队成员并原地更新 role |
| `uniai squad activity <issue-id> <action\|no_action\|failed> --reason "..."` | 队长智能体每轮结束时调用，记录 evaluation |

完整模型见 [小队](/squads)。

## Autopilots

| 命令 | 用途 |
|---|---|
| `uniai autopilot list` | 列出工作区所有 autopilot |
| `uniai autopilot get <id>` | 查看单个 autopilot |
| `uniai autopilot create ...` | 创建 autopilot |
| `uniai autopilot update <id> ...` | 修改 |
| `uniai autopilot delete <id>` | 删除 |
| `uniai autopilot runs <id>` | 查看运行历史 |
| `uniai autopilot trigger <id>` | 手动触发一次 |
| `uniai autopilot trigger-add/update/delete <id>` | 管理 schedule 或 webhook trigger |
| `uniai autopilot trigger-rotate-url <id> <trigger-id>` | 轮换 webhook trigger URL |

`uniai autopilot create/update` 还支持 `--subscriber`，为 `create_issue` 模式创建的 issue 设置默认订阅人；`update` 支持 `--clear-subscribers` 清空默认订阅人。

## 守护进程和运行时

| 命令 | 用途 |
|---|---|
| `uniai daemon start` | 启动 daemon（默认后台；加 `--foreground` 前台跑）|
| `uniai daemon stop` | 停止 daemon |
| `uniai daemon restart` | 重启 daemon |
| `uniai daemon status` | 查看 daemon 是否在线 + 并发情况 |
| `uniai daemon logs` | 查看 daemon 日志 |
| `uniai runtime list` | 列出当前工作区的 runtime |
| `uniai runtime usage` | 查看资源使用情况 |
| `uniai runtime activity` | 近期活动记录 |
| `uniai runtime update <id> ...` | 在某个 runtime 上触发 CLI 更新 |
| `uniai runtime delete <id> [--cascade]` | 删除 runtime，可选择同时归档绑定的智能体 |
| `uniai runtime profile ...` | 管理自定义 runtime profile 和本机路径覆盖 |

| 命令 | 用途 |

| `uniai config` | 查看 / 修改 CLI 本地配置 |
| `uniai version` | 显示 CLI 版本 |

## 查完整 flag

每条命令都支持 `--help`：

```bash
uniai issue create --help
uniai agent update --help
```

v2 会给每条命令一个独立的详细 reference 页。
