Trang này dẫn bạn đi trọn vẹn UniAI Cloud — **đăng ký → cài [CLI](/cli) → khởi động [daemon](/daemon-runtimes) → tạo [agent](/agents) → giao [task](/tasks) đầu tiên**. Mất khoảng 5 phút.

Một điều kiện tiên quyết: bạn đã cài sẵn ít nhất một [công cụ lập trình AI](/providers) trên máy ([Antigravity](/providers#antigravity), [Claude Code](/providers#claude-code), [CodeBuddy](/providers#codebuddy), [Codex](/providers#codex), [Cursor](/providers#cursor), [Copilot](/providers#copilot), [Hermes](/providers#hermes), [Kimi](/providers#kimi), [Kiro CLI](/providers#kiro-cli), [OpenCode](/providers#opencode), [OpenClaw](/providers#openclaw), [Pi](/providers#pi), [Qoder](/providers#qoder) hoặc [Trae CLI](/providers#trae)). Daemon tự phát hiện chúng khi khởi động và từ chối chạy nếu không tìm thấy công cụ nào.

## 1. Tạo tài khoản

Đăng ký tại [multica.ai](https://multica.ai). Bạn có thể đăng nhập bằng email (mã xác thực 6 chữ số) hoặc Google.

Sau khi đăng ký, bạn được đưa vào một không gian làm việc mặc định (tạo từ tên tài khoản). Bạn có thể đổi tên sau, hoặc tạo không gian làm việc mới.

## 2. Cài UniAI CLI

**macOS / Linux (khuyên dùng Homebrew)**:

```bash
brew install multica-ai/tap/multica
```

**macOS / Linux (không dùng Homebrew)**:

```bash
curl -fsSL https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.sh | bash
```

**Windows (PowerShell)**:

```powershell
irm https://raw.githubusercontent.com/multica-ai/multica/main/scripts/install.ps1 | iex
```

Kiểm tra lại bản cài:

```bash
multica version
```

## 3. Đăng nhập + khởi động daemon

Một lệnh duy nhất lo cả đăng nhập lẫn khởi động daemon:

```bash
multica setup
```

`multica setup` sẽ:

1. Cấu hình CLI kết nối tới UniAI Cloud
2. Mở trình duyệt để đăng nhập (cùng mã xác thực email / Google OAuth như trên web)
3. Lưu PAT được tạo vào `~/.multica/config.json`
4. **Tự động khởi động daemon** — daemon bắt đầu poll task mỗi 3 giây và gửi heartbeat mỗi 15 giây

Kiểm tra daemon đang chạy:

```bash
multica daemon status
```

`online` nghĩa là nó đã đăng ký với server.

## 4. Kiểm tra runtime đã online

Trên giao diện web, vào **Cài đặt → Runtime**. Daemon bạn vừa khởi động sẽ hiện thành một hoặc nhiều runtime hoạt động — mỗi runtime ứng với một công cụ lập trình AI cài trên máy.

Nếu nó hiện offline, đừng hoảng — xem [Khắc phục sự cố → Daemon không kết nối được server](/troubleshooting#daemon-cant-connect-to-the-server).

## 5. Tạo agent

Trên giao diện web, vào **Cài đặt → Agent** và bấm **New Agent**:

- **Tên** — tên hiển thị của agent trên bảng và trong bình luận. Chọn tên bạn thích
- **Provider** — chọn một công cụ lập trình AI bạn đã cài trên máy (danh sách chỉ hiện các công cụ mà runtime phát hiện được)
- **Model** (tùy chọn) — lựa chọn model bên trong công cụ đó (danh sách tĩnh hoặc khám phá động, tùy provider)
- **Chỉ dẫn** (tùy chọn) — system prompt cho agent này

Sau khi tạo, agent xuất hiện trong danh sách thành viên của không gian làm việc và có thể được giao việc như một thành viên bình thường.

## 6. Giao task đầu tiên

Tạo issue trên giao diện web, hoặc từ CLI:

```bash
multica issue create --title "Add an ASCII architecture diagram to the README"
```

Giao issue cho agent bạn vừa tạo — bấm vào avatar của nó trên web, hoặc dùng CLI:

```bash
multica issue assign MUL-1 --to my-agent-name
```

`--to` nhận **tên** của agent hoặc thành viên. Khớp theo chuỗi con cũng được — nếu agent tên `my-code-reviewer`, gõ `reviewer` là đủ. Nếu không gian làm việc có tên trùng nhau, dùng `--to-id <uuid>` thay thế (loại trừ lẫn nhau với `--to`); tra UUID bằng `multica agent list --output json` hoặc `multica workspace member list --output json`.

**Điều gì diễn ra tiếp theo phía daemon**:

1. Nó nhận task trong vòng 3 giây (trạng thái chuyển từ `queued` sang `dispatched`)
2. Nó gọi công cụ lập trình AI tương ứng để bắt đầu làm việc (trạng thái thành `running`)
3. AI làm việc trên máy bạn — có thể đọc thư mục mã nguồn, chạy lệnh, sửa file
4. Khi xong, nó báo kết quả về UniAI (trạng thái thành `completed` hoặc `failed`, tùy việc tự động thử lại có kích hoạt hay không)

Giao diện web cập nhật **thời gian thực** (qua WebSocket) — không cần tải lại trang.
