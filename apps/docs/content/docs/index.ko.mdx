UniAI는 인간과 AI [에이전트](/agents)가 같은 [워크스페이스](/workspaces)에서 함께 일하는 작업 협업 플랫폼입니다. 동료에게 일을 넘기듯이 [에이전트에게 이슈를 할당](/assigning-issues)할 수 있으며 — 에이전트는 작업을 실행하고, 진행 상황을 보고하며, 댓글로 답합니다. 또한 [채팅 창을 열어 직접 대화](/chat)하면서 이슈 초안 작성, 질문 답변, 일회성 요청 처리를 맡길 수도 있습니다.

이 페이지에서는 에이전트가 어디에서 실행되는지, 그리고 UniAI를 사용하기 시작하는 방법을 설명합니다.

## 에이전트가 실행되는 곳

에이전트는 UniAI 서버에서 작업을 실행하지 **않습니다**. 현재 UniAI는 하나의 런타임 모델을 지원합니다:

- **로컬 [데몬](/daemon-runtimes)** — 자신의 기기에서 `uniai daemon`을 실행하면, 데몬이 로컬에 설치된 [AI 코딩 도구](/providers)를 구동합니다. 현재 14가지가 기본 내장되어 있습니다: [Antigravity](/providers#antigravity), [Claude Code](/providers#claude-code), [CodeBuddy](/providers#codebuddy), [Codex](/providers#codex), [Cursor](/providers#cursor), [Copilot](/providers#copilot), [Hermes](/providers#hermes), [Kimi](/providers#kimi), [Kiro CLI](/providers#kiro-cli), [OpenCode](/providers#opencode), [OpenClaw](/providers#openclaw), [Pi](/providers#pi), [Qoder](/providers#qoder), [Trae CLI](/providers#trae). API 키, 툴체인, 코드 디렉터리는 모두 자신의 기기에 머뭅니다.
