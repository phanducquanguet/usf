UniAI CLI は、Web UI でできるほぼすべての操作をそのまま提供します（[イシュー](/issues)の作成、[エージェント](/agents)の割り当て、[デーモン](/daemon-runtimes)の起動など）。このページでは、すべてのトップレベルコマンドを 1 行の説明とともに一覧します。フラグや例の完全な一覧は `uniai <command> --help` を実行してください。

## 認証する

CLI を初めて使うときにこのコマンドを実行して、**パーソナルアクセストークン（PAT）**を取得します。

```bash
uniai login
```

ブラウザが自動的に開きます。Web アプリで承認すると、CLI が PAT（`mul_` プレフィックス付き）を `~/.multica/config.json` に保存します。これ以降のすべてのコマンドはこの PAT で認証されます。

<Callout type="tip">
CI やヘッドレス環境では、ブラウザフローをスキップできます。Web アプリの **Settings → Personal Access Tokens** で PAT を作成し、`uniai login --token <mul_...>` で直接渡してください。
</Callout>

トークンの種類による違いについては、[認証とトークン](/auth-tokens)を参照してください。

## 認証とセットアップ

| コマンド | 用途 |
|---|---|
| `uniai login` | ログインして PAT を保存 |
| `uniai auth status` | 現在のログイン状態、ユーザー、ワークスペースを表示 |
| `uniai auth logout` | ローカルの PAT を削除 |
| `uniai setup cloud` | UniAI Cloud のワンショットセットアップ（ログイン + デーモンのインストール） |

## ワークスペースとメンバー

| コマンド | 用途 |
|---|---|
| `uniai workspace list` | アクセスできるすべてのワークスペースを一覧 |
| `uniai workspace get <slug>` | 1 つのワークスペースの詳細を表示 |
| `uniai workspace member list` | 現在のワークスペースのメンバーを一覧 |
| `uniai workspace update <id> --name "..." [--description "..."] [--context "..."] [--issue-prefix "..."]` | ワークスペースのメタデータを更新（admin/owner）。長いフィールドは `--description-stdin` / `--context-stdin` を使用できます。 |

## イシューとプロジェクト

<Callout type="info">
`list` 系のコマンド（`uniai issue list`、`autopilot list`、`project list` など）は、デフォルトで短く**そのままコピー＆ペーストできる** ID を出力します。イシューは `UNI-123` のようなイシューキー、それ以外のリソースは短い UUID プレフィックスです。以下の後続コマンドの `<id>` 引数は短い ID と完全な UUID のどちらも受け取るため、一般的な流れは `uniai issue list` → キーをコピー → `uniai issue get UNI-123` となります。正式な UUID が必要なときは `list` コマンドに `--full-id` を渡してください。
</Callout>

| コマンド | 用途 |
|---|---|
| `uniai issue list` | イシューを一覧（コピー＆ペーストできるイシューキーを出力） |
| `uniai issue get <id>` | 単一のイシューを表示（イシューキーまたは UUID を受け取る） |
| `uniai issue create --title "..."` | 新しいイシューを作成 |
| `uniai issue update <id> ...` | イシューを更新（ステータス、優先度、担当者など） |
| `uniai issue assign <id> --to <name>` | メンバー、エージェント、またはスクワッドに割り当て（エージェントへの割り当ては実行をトリガー） |
| `uniai issue status <id> <status>` | ステータス変更のショートカット |
| `uniai issue search <query>` | キーワード検索 |
| `uniai issue children <id>` | サブイシューを stage ごとに一覧 |
| `uniai issue pull-requests <id>` | 紐付いた pull request と状態を一覧 |
| `uniai issue runs <id>` | イシュー上のエージェント実行を表示 |
| `uniai issue run-messages <task-id>` | 1 回の実行メッセージを表示 |
| `uniai issue usage <id>` | イシュー単位の集計 token 使用量を表示 |
| `uniai issue rerun <id>` | イシューの現在のエージェント担当者向けに新しいタスクを再キューイング |
| `uniai issue cancel-task <task-id>` | キュー中または実行中のタスクをキャンセル |
| `uniai issue comment <id> ...` | ネスト: コメントの表示 / 投稿 |
| `uniai issue comment resolve/unresolve <comment-id>` | コメントスレッドを解決済み / 未解決にする |
| `uniai issue subscriber <id> ...` | ネスト: 購読 / 購読解除 |
| `uniai issue metadata <id> ...` | ネスト: イシューメタデータの読み書き |
| `uniai issue label <id> ...` | ネスト: イシューのラベルを管理 |
| `uniai project list/get/create/update/delete/status` | プロジェクトの CRUD |
| `uniai label list/create/update/delete` | ワークスペースラベルの CRUD |

## エージェントとスキル

| コマンド | 用途 |
|---|---|
| `uniai agent list` | ワークスペースのエージェントを一覧 |
| `uniai agent get <slug>` | エージェントの構成を表示 |
| `uniai agent create ...` | エージェントを作成 |
| `uniai agent update <slug> ...` | エージェントを更新 |
| `uniai agent archive <slug>` | アーカイブ |
| `uniai agent restore <slug>` | アーカイブ済みのエージェントを復元 |
| `uniai agent tasks <slug>` | エージェントのタスク履歴を表示 |
| `uniai agent avatar <slug>` | エージェントのアバターをアップロード |
| `uniai agent env <slug> ...` | エージェントの custom environment variables を読み取り / 置換 |
| `uniai agent skills ...` | ネスト: スキルのアタッチ / デタッチ |
| `uniai skill list/get/create/update/delete` | スキルの CRUD |
| `uniai skill import ...` | GitHub、ClawHub、またはローカルマシンからスキルをインポート |
| `uniai skill files ...` | ネスト: スキルのファイルを管理 |

### スキルインポートの競合

`uniai skill import --url <url>` の既定値は `--on-conflict fail` です。同じ名前のスキルがすでに存在する場合、コマンドは構造化された `conflict` 結果で終了し、ワークスペースは変更されません。

既存スキルの作成者で、スキル ID とエージェントの紐付けを維持したまま内容を置き換える場合は `--on-conflict overwrite` を使います。既存スキルを残してコピーを取り込む場合は `--on-conflict rename` を使うと、`-2` のような接尾辞が自動で付きます。同名の項目を単に飛ばす場合は `--on-conflict skip` を使います。

```bash
uniai skill import --url https://skills.sh/acme/repo/review-helper
uniai skill import --url https://skills.sh/acme/repo/review-helper --on-conflict overwrite
uniai skill import --url https://skills.sh/acme/repo/review-helper --on-conflict rename
uniai skill import --url https://skills.sh/acme/repo/review-helper --on-conflict skip
```

## スクワッド

| コマンド | 用途 |
|---|---|
| `uniai squad list` | ワークスペースのスクワッドを一覧 |
| `uniai squad get <id>` | 単一のスクワッドを表示 |
| `uniai squad create --name "..." --leader <agent>` | スクワッドを作成（owner / admin） |
| `uniai squad update <id> ...` | 名前、説明、指示、リーダー、またはアバターを更新 |
| `uniai squad delete <id>` | アーカイブ（ソフト削除） — 割り当て済みのイシューをリーダーに移管 |
| `uniai squad member list/add/remove <squad-id>` | スクワッドメンバーを管理 |
| `uniai squad activity <issue-id> <action\|no_action\|failed> --reason "..."` | スクワッドリーダーエージェントがターンごとに評価を記録するために使用 |

完全なモデルについては[スクワッド](/squads)を参照してください。

## オートパイロット

| コマンド | 用途 |
|---|---|
| `uniai autopilot list` | ワークスペースのすべてのオートパイロットを一覧 |
| `uniai autopilot get <id>` | 単一のオートパイロットを表示 |
| `uniai autopilot create ...` | オートパイロットを作成 |
| `uniai autopilot update <id> ...` | 更新 |
| `uniai autopilot delete <id>` | 削除 |
| `uniai autopilot runs <id>` | 実行履歴を表示 |
| `uniai autopilot trigger <id>` | 手動で実行をトリガー |
| `uniai autopilot trigger-add/update/delete <id>` | schedule または webhook trigger を管理 |
| `uniai autopilot trigger-rotate-url <id> <trigger-id>` | webhook trigger URL をローテート |

`uniai autopilot create/update` は、`create_issue` モードの autopilot が作成するイシューの既定購読者を設定する `--subscriber` も受け取ります。`update` では `--clear-subscribers` で削除できます。

## デーモンとランタイム

| コマンド | 用途 |
|---|---|
| `uniai daemon start` | デーモンを起動（デフォルトはバックグラウンド。`--foreground` を追加するとフォアグラウンドで実行） |
| `uniai daemon stop` | デーモンを停止 |
| `uniai daemon restart` | デーモンを再起動 |
| `uniai daemon status` | デーモンがオンラインかどうかと同時実行数を確認 |
| `uniai daemon logs` | デーモンのログを表示 |
| `uniai runtime list` | 現在のワークスペースのランタイムを一覧 |
| `uniai runtime usage` | リソース使用量を表示 |
| `uniai runtime activity` | 最近のアクティビティログ |
| `uniai runtime update <id> ...` | ランタイム上で CLI 更新を開始 |
| `uniai runtime delete <id> [--cascade]` | ランタイムを削除し、必要なら紐付くエージェントもアーカイブ |
| `uniai runtime profile ...` | カスタム runtime profile とローカルパス上書きを管理 |

## その他

## 完全なフラグを確認する

すべてのコマンドが `--help` をサポートしています。

```bash
uniai issue create --help
uniai agent update --help
```

v2 では、各コマンドごとに専用の詳細なリファレンスページを提供する予定です。
